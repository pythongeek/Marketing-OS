### 5.1 Pipeline Architecture

```
┌──────────────────────────────────────────────────────────┐
│ STAGE 1: KEYWORD RESEARCH (DataForSEO + Ahrefs API)      │

                            ↓
┌──────────────────────────────────────────────────────────┐
│ STAGE 2: DATA ENRICHMENT (Python ETL)                    │
│   ↓ Pulls: Search volume, SERP features, industry stats  │
│   ↓ Merges: First-party conversion data                  │

                            ↓
┌──────────────────────────────────────────────────────────┐
│ STAGE 3: TEMPLATE POPULATION (LLM + structured data)     │
│   ↓ LLM fills templates with data variables              │

                            ↓

│   ↓ Modular prompts for each section                    │
│   ↓ RAG-augmented with product docs + case studies       │

                            ↓
┌──────────────────────────────────────────────────────────┐

                            ↓
┌──────────────────────────────────────────────────────────┐

│   ↓ Auto-deploy via CI/CD                               │

                            ↓

│   ↓ Performance tracking, refresh triggers               │

```

### 5.2 LLM Prompt Architecture (Modular)

```yaml
section_hero:
  inputs: [industry, value_prop, primary_keyword, cta_text]
  prompt_template: |
    Write a 150-character H1 and 200-character subheadline 
    for an AI marketing automation product targeting {industry}. 
    Primary keyword: {primary_keyword}. 
    Tone: authoritative, ROI-focused, plainspoken. 
    Include the keyword naturally in the H1.
  
section_pain_points:
  inputs: [industry, pain_points_list]
  prompt_template: |
    Write 3-5 pain point sections for {industry} marketing leaders. 
    For each, include: bold headline, 50-word description, 
    a quantified cost or impact statement. 
    Pain points: {pain_points_list}
    Reference at least 2 data points from the inputs.

section_workflow:
  inputs: [industry, workflow_steps]
  prompt_template: |
    Describe a 4-6 step marketing automation workflow for {industry}. 
    Format: numbered steps with tool/feature name + 25-word description.
    Steps: {workflow_steps}
    Include 1 sentence on the expected outcome per step.
```
