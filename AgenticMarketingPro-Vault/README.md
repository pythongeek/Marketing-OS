# AgenticMarketingPro Vault

> The primary memory and operating system for AgenticMarketingPro — an AI-native digital marketing agency. This Obsidian vault is the single source of truth for every client, every agent, every campaign, and every decision.

## What this is

This is not a documentation site. This is a **live, continuously-updated knowledge graph** that powers an agent operating system. Every markdown file is:
- **Queryable** by Dataview (live dashboards inside Obsidian)
- **Embeddable** by the RAG pipeline (Chroma + LlamaIndex → context for every agent call)
- **Versioned** in Git (full audit trail of every change)
- **Tagged** with a strict taxonomy (faceted queries, no chaos)

## How to use this vault

### For the human strategist (you)
1. **Open Obsidian** → open this folder as a vault
2. **Install recommended plugins** (Dataview, Templater, Obsidian Git, Smart Connections) — see `.obsidian/community-plugins.json`
3. **Start each morning** at `00-Agency-Core/_dashboards/command-center.md` — your live agency dashboard
4. **Review** `11-Ops/agent-task-queue.md` — what's on the docket today
5. **Approve** anything in `11-Ops/agent-task-queue.md` marked "needs approval"
6. **End each day** by reviewing `11-Ops/daily-ops-log.md` (auto-generated by Atlas)

### For the agent OS
Every agent:
1. **Reads** relevant context from the vault via RAG retrieval (BM25 + Chroma vector + Kùzu graph)
2. **Writes** outputs to declared vault paths only (enforced at n8n orchestration layer)
3. **Logs** every run to `11-Ops/agent-logs/[agent-name]/`
4. **Triggers** Git commit with `[agent-name]: [task summary]` format
5. **Updates** relevant Dataview dashboards automatically

## Setup instructions (one-time)

### 1. Open in Obsidian
1. Install [Obsidian](https://obsidian.md/) (free)
2. Open this folder as a vault: `File → Open vault → Open folder as vault`
3. Allow community plugins (Settings → Community plugins → Turn on community plugins)
4. Install plugins: Dataview, Templater, Obsidian Git, Smart Connections, Calendar

### 2. Connect to Git
1. Create a private GitHub repo (or GitLab, Bitbucket private)
2. Initialize this vault as a Git repo: `git init && git remote add origin [your-repo-url]`
3. Configure Obsidian Git plugin with your credentials
4. Set auto-commit + auto-pull intervals (5 minutes recommended)

### 3. Configure secrets
1. Install [1Password CLI](https://developer.1password.com/docs/cli/) or HashiCorp Vault
2. Add API credentials for each integration in `11-Ops/integrations/`
3. Each integration file references credentials by name (e.g., `${AHREFS_API_KEY}`) — never store credentials in the vault itself

### 4. Stand up the RAG pipeline
1. Install Chroma DB: `pip install chromadb`
2. Install LlamaIndex: `pip install llama-index`
3. Configure OpenAI API key for embeddings (text-embedding-3-small)
4. Run the ingestion script (TBD — will be in `11-Ops/scripts/ingest.py`)
5. Verify with sample query: "what do we know about Acme Corp?"

### 5. Stand up n8n
1. Self-host n8n via Docker (recommended: Hetzner CX31 VPS, $8/mo)
2. Configure webhook URLs for each agent
3. Set up daily ops loop workflow (9-step Atlas cycle)
4. Test with a single agent run (e.g., daily GSC pull)

## Conventions

### Frontmatter
Every note has YAML frontmatter. See `11-Ops/playbooks/frontmatter-standards.md` for the full type registry.

### Tags
Tags follow `category/value` pattern. See `11-Ops/playbooks/tag-dictionary.md`.

### Wikilinks
Use `[[note-name]]` for internal links. The RAG pipeline converts these into graph edges.

### Git commits
Format: `[agent-name]: [verb] [object]` — e.g., `[content-strategist]: generate brief for ai-compliance-saas`
