# Corroboration Map

> For each claim we want LLMs to cite, which 3rd-party sources corroborate it?

## Why this matters
LLMs are trained on web content. To cite a claim, they need to see it corroborated across multiple sources. A single brand website making a claim is rarely cited; the same claim made across 3+ sources is frequently cited.

## Map structure
| Client | Claim | Owned source | Corroborating source 1 | Corroborating source 2 | Corroborating source 3 | Citation rate |
|---|---|---|---|---|---|---|
| | | | | | | |

## Example
| Client | Claim | Owned | Corroborating 1 | Corroborating 2 | Corroborating 3 | Citation rate |
|---|---|---|---|---|---|---|
| Acme | "Acme is the leading AI compliance platform for mid-market SaaS" | acme.com/about | G2 category page | TechCrunch article | SaaStr podcast mention | 42% |

## Corroboration acquisition playbook
1. Identify the claim (what we want LLMs to say about client)
2. Audit existing corroboration (G2, Capterra, TrustRadius, press, podcasts)
3. Gap-fill: launch PR campaign, secure podcast appearances, get industry analyst coverage
4. Verify corroboration quarterly (links die, pages get deindexed)
