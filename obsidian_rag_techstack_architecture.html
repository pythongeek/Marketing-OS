RAG pipeline design
Every `.md` file in the vault is chunked, embedded, and indexed into a vector DB. The Orchestrator Agent retrieves relevant context before every task — so it always knows your clients, goals, and current state.
Live sync strategy
Agents write back to the vault after every task. GSC Agent updates `gsc-weekly-log.md` automatically. Competitor Agent overwrites `competitor-map.md`. The vault is always the single source of truth.
Obsidian tagging system
Every note tagged by: `#client/[name]`, `#priority/high`, `#agent/[name]`, `#status/done`. Dataview plugin turns tags into live dashboards inside Obsidian itself — no extra tooling needed.
Version control
Vault lives in a private Git repo. Every agent write triggers a commit with `[agent-name]: [task summary]` as the commit message. Full audit trail of every decision the OS makes.

RAG pipeline — how context flows into every agent call
Obsidian vault .md files → LlamaIndex chunker (512 tokens, 50 overlap) → text-embedding-3-small → Chroma / Pinecone index
n8n trigger fires (schedule / event) → Task intent identified → Top-k retrieval (k=8–15 chunks) → Context injected into agent system prompt
Claude API call (with context + task) → Agent output (content / data / action) → Write-back to Obsidian → Re-embed updated note
External API (GSC / Ahrefs / GA4) → Data normalised to .md → Vault update → RAG index refreshed nightly

Agent system prompt architecture — the most critical design decision
Every agent prompt has 5 fixed layers
1. Role definition — who the agent is, what it owns, what it never does.

2. RAG context block — injected at runtime: relevant vault chunks pulled by LlamaIndex retrieval.

3. Toolset declaration — which APIs this agent can call, which vault folders it can write to.

4. Output format spec — exact markdown structure the agent must return (for vault write-back).

5. Escalation rule — when to flag for human review vs. proceed autonomously.
Human approval gates — what always needs a human
— Any ad spend change > $100/day
— Publishing content to client sites
— Sending any outreach email
— Changing site technical structure (redirects, canonicals)
— Any action affecting a live Google Ads campaign
— New programmatic SEO batch > 50 pages

Everything else: the OS runs fully autonomously and logs to `daily-ops-log.md`.
